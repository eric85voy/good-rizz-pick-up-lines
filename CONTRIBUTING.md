# Contributing to Good Rizz Pick Up Lines

Thank you for your interest in contributing to this project!

## Contribution Guidelines

We believe in maintaining clean, readable, and scalable code. Please ensure:
- Code follows consistent formatting.
- Components are reusable and modular.
- No offensive or inappropriate pickup lines are added.
- Proper comments are included where necessary.

## How to Contribute

1. Fork the repository.
2. Create a new branch for your changes.
3. Follow the project's coding standards.
4. Commit your changes and push to your fork.
5. Open a pull request to the main branch.

Refer to this file for detailed contribution instructions.

## Reporting Issues

If you encounter a bug:
- Check existing issues first.
- Provide a detailed explanation.
- Include screenshots if applicable.
- Mention steps to reproduce the issue.

## Suggesting Features

Have an idea to improve the rizz experience?
- Open a feature request.
- Clearly describe the enhancement.
- Explain the benefit to users.

## Code of Conduct

- Be respectful and professional.
- No hate speech or harassment.
- Keep contributions positive and helpful.
- Constructive feedback only.

## Review Process

- Pull requests are reviewed within 2–5 business days.
- Code quality and functionality are checked.
- Contributors may be asked for revisions.
- Once approved, changes are merged.
